# Сериализация двусвязного списка с произвольными ссылками

## Описание задачи

Реализована сериализация и десериализация двусвязного списка `ListNode`, где:
- `prev` и `next` указывают на предыдущий/следующий элемент списка
- `rand` указывает на произвольный элемент этого же списка или `nullptr`
- Каждый элемент содержит строку `data`

Программа читает текстовое описание списка из файла `inlet.in`, строит связанный список и сохраняет его в бинарный файл `outlet.out`.

## Структура узла списка

struct ListNode {
    ListNode* prev = nullptr;  // указатель на предыдущий элемент
    ListNode* next = nullptr;  // указатель на следующий элемент
    ListNode* rand = nullptr;  // указатель на произвольный элемент
    std::string data;          // пользовательские данные
};

## Формат входного файла

Текстовый файл, где каждая строка описывает один узел в формате:
<data>;<rand_index>
data - строка произвольного содержания (может содержать пробелы и спецсимволы)
rand_index - индекс узла (начиная с 0), на который указывает rand, либо -1, если rand = nullptr

## Пример входного файла (inlet.in):

apple;2
banana;-1
carrot;1

## Алгоритм работы

1.Чтение из текстового файла (readFromTextFile)
  1.1 Открытие файла inlet.in
  1.2 Построчное чтение и парсинг каждой строки:
    -Разделение строки по символу ';'
    -Сохранение data и rand_index
  1.3 Создание узлов списка в динамической памяти
  1.4 Установка связей:
    -prev и next на основе порядка в списке
    -rand на основе индексов из файла
  1.5 Возврат указателя на голову списка
2. Сохранение в бинарный файл (saveToBinaryFile)
  2.1 Открытие файла outlet.out в бинарном режиме
  2.2 Обработка пустого списка (запись 0 узлов)
  2.3 Сбор всех узлов в вектор для удобства навигации
  2.4 Вычисление позиций каждого узла в файле
  2.5 Запись каждого узла:
    -Поиск позиций связанных узлов
    -Запись трех позиций (prev, next, rand)
    -Запись строки данных (размер + содержимое)
3. Обработка ошибок
  Программа обрабатывает следующие ошибки:
    -Невозможно открыть входной файл
    -Невозможно создать выходной файл
    -Некорректные строки в входном файле (пропускаются)
    -Все ошибки выводятся в stderr, программа завершается с кодом возврата 1.

Особенности реализации

Безопасность памяти: Все выделенные узлы освобождаются в методе deleteList
Поддержка UTF-8: Строки сохраняются в бинарном виде без изменения кодировки
Масштабируемость: Поддерживается до 10⁶ узлов
Простота кода: Используются только базовые конструкции C++ для наглядности

Ограничения

Максимальное число узлов: 1 000 000
Максимальная длина строки data: 1000 символов
Размер выходного файла: примерно (24 + средняя длина строки) * количество узлов байт
